# 2026-04-28 -- vBRIEF `x-<consumer>/` extension namespace (proposal pointer)

Upstream RFC filed at [`deftai/vBRIEF#12`](https://github.com/deftai/vBRIEF/issues/12). Proposes ratification of a reserved `^x-[a-z0-9-]+/` extension-properties namespace, a normative round-trip-preservation reader contract, and a forward commitment that future core property names will not match the pattern. Non-breaking; targets v0.6.1 or folding into v0.7.

Drafted in coordination with the [`deftai/directive#711`](https://github.com/deftai/directive/issues/711) analysis. Companion directive-side work tracked at:

- [`deftai/directive#714`](https://github.com/deftai/directive/issues/714) -- docs(vbrief): document directive as the vBRIEF reference consumer + four-tier extensibility model.
- [`deftai/directive#715`](https://github.com/deftai/directive/issues/715) -- feat(vbrief): conformance fixture set + round-trip preservation in validator.

Original analysis comment on the catalyst proposal: [`deftai/directive#711` comment-4337985399](https://github.com/deftai/directive/issues/711#issuecomment-4337985399).
